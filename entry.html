Network & script inspection tool

  1. Open the target site in this browser (same window). You may use a new tab (Ctrl+T) for the site; capture and the output folder name use the last tab with a normal https:// URL.
  2. Use the site as usual (log in, click around, load the pages you care about). All requests and script responses are recorded.
  3. When finished, switch to the terminal and press Enter to save and close.

Output is written under output/<hostname>/:

  • networkcalls.json – all requests and responses
  • scripts.json – JavaScript responses only

Session is saved to session.json so the next run can reuse cookies/storage. Use --no-session to start fresh.

Getting reCAPTCHA / “noCAPTCHA” errors on LinkedIn or Google? Use a persistent Chrome profile so the site sees a real browser with history and a stable fingerprint. Copy your profile once, then pass it on every run:

cp -r ~/.config/google-chrome ./chrome-profile
python main.py --user-data-dir chrome-profile

Other options: --url https://example.com forces output hostname  ·  --domains example.com,api.example.com limits capture  ·  --cdp-debugger dumps eval’d scripts (slow).
